# Requerimientos del Sistema de Eventos Calendarizados

## Requerimiento funcional

### RF-01: Visualización de eventos universitarios

El sistema debe permitir a los estudiantes y profesores visualizar todos los eventos universitarios calendarizados, incluyendo clases, reuniones, actividades académicas y otros eventos institucionales.

#### Criterios de aceptación

1. El usuario puede ver una lista de eventos disponibles.
2. Cada evento debe mostrar nombre, fecha, hora y lugar.
3. El usuario puede identificar si un evento pertenece a clases, actividades o reuniones.
4. El sistema debe mostrar eventos actualizados cuando se registren cambios.

---

## Requerimiento no funcional

### RNF-01: Facilidad de uso

El sistema debe ser fácil de utilizar para estudiantes y profesores, sin necesidad de capacitaciones, entrenamientos o manuales de uso.

#### Criterios de aceptación

1. El usuario debe poder acceder al calendario en menos de 3 clics.
2. La interfaz debe ser clara, ordenada e intuitiva.
3. Los botones y opciones principales deben ser fáciles de identificar.
4. Un usuario nuevo debe poder consultar un evento sin ayuda externa.
